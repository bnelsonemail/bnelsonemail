## 🧠 Engineering Philosophy

Engineering has always been about making informed decisions with imperfect information.

Whether designing a water distribution system, evaluating a lift station, or planning future infrastructure investments, engineers rarely have complete data. Success comes from combining technical knowledge, engineering judgment, and the best information available at the time.

That philosophy guides every project in this portfolio.

I don't view machine learning as a replacement for engineering expertise. Instead, I see it as another engineering tool that can help identify patterns, quantify uncertainty, and support better decision-making.

Likewise, software engineering is more than writing code. Well-designed software makes engineering knowledge repeatable, scalable, and accessible to others.

My goal is to build practical engineering software that transforms data into information and information into better infrastructure decisions.

### 🚰 Toronto Pipeline Failure Prediction — Applied ML & Infrastructure Risk System

[Toronto Pipeline Failure Predict](https://github.com/Brice-Machine-Learning/Toronto-Pipeline-Failure-Predict)

A real-world **applied machine learning project** focused on predicting **water main failure risk** using public infrastructure, environmental, and asset data from the City of Toronto. The project is intentionally designed as the foundation for a deployable **infrastructure analytics platform**, not a standalone model.

This work prioritizes **data realism**, **engineering constraints**, and **defensible modeling decisions**, reflecting the realities faced by municipal utilities where data is incomplete, heterogeneous, and operationally constrained.

**Status:** 🟢 Active Development
**Primary Competencies:** Applied Machine Learning • Infrastructure Analytics • Feature Engineering • Risk Modeling • Data Engineering

#### Applied ML Focus

* Asset-level feature engineering (installation year, material, diameter, environmental exposure)
* Spatial and temporal aggregation of infrastructure and failure data
* Failure labeling under incomplete and inconsistent municipal records
* Model evaluation aligned with risk prioritization, not academic metrics
* Explicit handling of uncertainty and data limitations

#### Systems & Backend Direction (In Progress / Planned)

* Backend service to operationalize risk scoring and model inference
* Secure authentication and role-based access for internal and stakeholder users
* API endpoints for asset-level risk queries and scenario analysis
* Dashboard integration for infrastructure risk visualization (Power BI or equivalent BI tooling)
* Deployment-focused design emphasizing reproducibility, auditability, and model versioning

#### Engineering Philosophy

* ML treated as a system component, not a notebook artifact
* Clear separation between data processing, modeling logic, and serving layers
* Design decisions driven by how results will be consumed by engineers, planners, and decision-makers

**Goal** 
>Deliver an applied ML–driven infrastructure risk system that bridges modeling, backend services, and operational dashboards—demonstrating end-to-end thinking from raw municipal data to deployable decision support.

**Long-Term Vision**
>Develop a production-ready infrastructure analytics platform that enables utilities to prioritize asset replacement, quantify infrastructure risk, and support engineering decision-making through applied machine learning and modern software engineering.

### 📡 Suspicious Sensor — Probabilistic Infrastructure Analytics

[Suspicious Sensor](https://github.com/Brice-Engineering-Projects/suspicious-sensor)

A probabilistic infrastructure analytics project focused on investigating anomalous pressure sensor behavior within municipal water distribution systems. The project combines statistical modeling, anomaly detection, data quality assessment, and engineering judgment to distinguish sensor faults from legitimate system events.

Rather than treating every anomalous reading as a system failure, this project explores how uncertainty can be quantified and incorporated into engineering decision-making.

**Status:** 🟢 Active Development

**Primary Competencies:** Infrastructure Analytics • Statistical Modeling • Anomaly Detection • Probabilistic Analysis • Data Engineering

#### Current Capabilities

- Synthetic infrastructure dataset development
- Data validation and quality assessment
- Exploratory data analysis
- Statistical characterization of pressure behavior
- Engineering-focused anomaly investigation
- Reproducible analytical workflows

#### Planned Development

- Probabilistic anomaly detection
- Bayesian inference for event classification
- Time-series analysis
- Multi-sensor correlation
- Risk scoring
- Interactive engineering dashboards

#### Engineering Focus

- Combining engineering judgment with statistical analysis
- Distinguishing instrumentation faults from legitimate hydraulic events
- Evaluating uncertainty rather than relying on deterministic thresholds
- Developing analytical workflows suitable for real-world utility operations

**Goal** 
>Develop an infrastructure analytics platform capable of assisting utilities in identifying, prioritizing, and investigating abnormal system behavior while reducing false alarms through probabilistic modeling and engineering context.
